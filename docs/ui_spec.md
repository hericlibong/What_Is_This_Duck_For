# UI Specification — WHAT IS THIS DUCK FOR?

## Core Principle

Single-page interface centered on:

* the question
* the duck
* the five analysis modes
* the report panel

The interface must feel like a **serious inquiry into a trivial object**.

---

## Visual Hierarchy (Top to Bottom)

1. Main Title
2. Duck (central visual object)
3. Action Buttons
4. Report Panel
5. Minimal Navigation

---

## Header (Lightweight)

* Keep it minimal
* Small contextual label allowed:
  "Department of Non-Essential Artifacts"
* Add a simple navigation link:

  * About

The header must NOT dominate the page.

---

## Main Title

Primary title (centered):

WHAT IS THIS DUCK FOR?

Optional subtitle (small, secondary):

A Serious Inquiry Into the Function of a Rubber Duck

---

## Duck Area

* Duck remains centered
* Treated as an object under inspection
* Light visual framing allowed (circle, halo, diagram feel)
* Avoid heavy sci-fi styling

---

## Buttons (5 Modes)

Use EXACT labels:

* ANALYZE THE DUCK (primary button)
* REQUEST DEEPER INTERPRETATION
* ESCALATE TO MINISTRY
* TRUST THE DUCK
* DO NOT TRUST THE DUCK

### Layout

* One primary button (ANALYZE)
* Four secondary buttons arranged in a simple grid (2x2)

No complex layout.

---

## Navigation

Minimal only:

* About

No full navbar, no multi-page system for now.

---

## Report Panel

Title:

MINISTERIAL ANALYSIS REPORT

Content structure:

* CASE ID
* STATUS
* CONFIDENCE LEVEL

Then the generated content.

Style:

* clean
* readable
* document-like

---

## Visual Tone

* Clean, structured
* Slight bureaucratic feel
* Paper / report aesthetic
* Duck is the only strong color accent

---

## Constraints

* No redesign into dashboard
* No heavy UI system
* No complex navigation
* No animation work yet
* No changes to data flow or backend

---

## Goal of This Phase

Prepare a clean, readable, focused interface that highlights:

* the question
* the duck
* the result

Nothing more.
